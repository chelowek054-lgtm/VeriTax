# VeriTax

Локальное приложение: вы кладёте выгрузку чата из Telegram в `.data/`, запускаете индексацию с прогрессом, задаёте вопрос и получаете ответ от LLM только по найденным веткам с указанием источников.

## Что нужно установить

1. **Ollama на компьютере** — эмбеддинги и чат всегда идут в локальный Ollama, не в Docker.
2. **Docker Desktop** — если хотите поднимать `Qdrant` и `FastAPI` одной командой. Либо можно запускать приложение через `pip` и `uvicorn` локально.

### Модели Ollama

В терминале на машине, где крутится Ollama:

```bash
ollama pull bge-m3
ollama pull qwen2.5:3b
```

## Запуск

### 1. Настройка `.env`

```text
copy .env.example .env
```

При `**docker compose**` адрес Ollama для контейнера приложения зашит в `docker-compose.yml`: `http://host.docker.internal:11434` (и `extra_hosts: host-gateway` для Linux). Значение `OLLAMA_BASE_URL` из `.env` не должно подменять его на `http://localhost:11434`, потому что внутри контейнера `localhost` — это не ваш ПК.

Для запуска **без Docker** (`uvicorn` на хосте) в `.env` используйте `OLLAMA_BASE_URL=http://127.0.0.1:11434`.

### 2. Данные чатов

Папка `**.data`** в корне проекта. Внутрь кладите **отдельной папкой** каждую выгрузку Telegram с файлом `result.json`.

Рядом с данными приложение сохраняет служебный файл `**.data/.veritax-index-state.json`**: в нём хранится информация, какие экспорты уже индексировались, когда именно и не изменился ли `result.json` после индексации.

#### Как выгрузить тему или чат из Telegram Desktop

1. Откройте нужную тему или чат.
2. Нажмите `Ещё`.
3. Выберите `Экспорт истории темы`.
4. В окне экспорта выберите `**Машиночитаемый JSON**`.
5. После экспорта скопируйте папку с `result.json` в `.data/`.

### 3. Docker: Qdrant + приложение

```bash
docker compose up --build
```

Перед этим запустите **Ollama** на хосте, чтобы она слушала порт `11434`.

Хранилище Qdrant лежит в папке проекта `**volumes/qdrant`**, а не во внутреннем docker named volume. Это упрощает перенос и бэкап данных.

Откройте [http://localhost/](http://localhost/) или порт из `FASTAPI_PORT` в `.env`.

- **Индексация**: кнопка на главной запускает фоновую задачу; статус и прогресс обновляются через polling (`POST /api/v1/index/jobs`, `GET /api/v1/index/jobs/{id}`).
- **Вопрос**: форма вызывает `POST /api/v1/ask`; ответ и раскрываемые источники показываются на странице.

### Синхронная индексация (API)

По-прежнему доступен блокирующий вызов `POST /api/v1/index` без прогресса — для скриптов и CLI.

## Переменные окружения


| Переменная                                 | Назначение                                                                          |
| ------------------------------------------ | ----------------------------------------------------------------------------------- |
| `OLLAMA_BASE_URL`                          | Базовый URL Ollama (`host.docker.internal` из контейнера или `127.0.0.1` локально). |
| `RAG_TOP_K`                                | Сколько веток забирать из Qdrant (по умолчанию 5).                                  |
| `ASK_CONTEXT_MAX_CHARS`                    | Лимит символов контекста для промпта.                                               |
| `OLLAMA_EMBED_MODEL` / `OLLAMA_CHAT_MODEL` | Модели эмбеддингов и чата.                                                          |


## Для разработчиков

- Зависимости: `pip install -r requirements.txt`
- Тесты: `pytest`
- Индексация из CLI: `python -m app.cli index`
- Цель продукта: [docs/MVP.md](docs/MVP.md)

## Ограничения MVP

- Реестр задач индексации хранится **в памяти процесса** — при нескольких воркерах или перезапуске контейнера `job_id` из UI перестанет находиться.
- Большие ветки обрезаются по `THREAD_TEXT_MAX_CHARS` при индексации и по `ASK_CONTEXT_MAX_CHARS` при ответе.
